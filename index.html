SQL pour les vrais débutants

Tu n’as jamais vu une base de données ? À la fin de ce cours tu seras capable de créer, remplir et utiliser une base complète.

1. Une base de données, c’est quoi ?
Une base de données = un gros tableau Excel, mais qui ne plante jamais, qui accepte des millions de lignes, et où tu peux chercher n’importe quoi en 0,01 seconde.

Dans la vraie vie : Facebook, Netflix, ta banque, Amazon → tout tourne avec des bases de données.
Imagine un fichier Excel avec 3 onglets :
• Utilisateurs → pseudo, email
• Blagues → question, réponse, qui l’a écrite
• Likes → qui a liké quelle blague
→ C’est exactement une base de données SQL.
2. Les 2 grandes familles (c’est tout ce que tu dois savoir)
SQL (relationnel) → des tableaux bien rangés avec des règles strictes. Parfait pour l’argent, les utilisateurs, les commandes.
NoSQL → des fichiers JSON souples. Parfait pour les messages, les logs, les trucs qui changent souvent.
Pour 95 % des projets sérieux → on utilise du SQL.
On va donc apprendre SQL (MySQL/MariaDB/PostgreSQL → c’est presque pareil).

6. Comment on conçoit une base (en 3 dessins)
MCD → dessin avec des boîtes (utilisateur, blague) et des flèches (qui écrit quoi).
MLD → on transforme le dessin en vraies tables avec des id.
UML → même chose mais plus orienté programmation.
Pour notre projet blagues : 
Utilisateur (id, pseudo)
Blague (id, question, réponse, auteur_id → lien vers utilisateur)
Like (utilisateur_id, blague_id)
Tu n’as besoin que de ça pour 99 % des projets.
7. Les types que tu vas utiliser tout le temps
INT → nombre entier (id, âge)
VARCHAR(255) → texte court (pseudo, email)
TEXT → texte long (description)
DATE / DATETIME → date et heure
TINYINT(1) → vrai/faux (0 ou 1)
AUTO_INCREMENT → le nombre augmente tout seul
8. Projet complet : Blagues en Folie (tout faire en 5 minutes)
Copie-colle tout ça d’un coup dans ton terminal SQL : 
CREATE DATABASE blagues_en_folie CHARACTER SET utf8mb4;
USE blagues_en_folie;
-- 1. Table utilisateurs
CREATE TABLE utilisateurs (
  id INT AUTO_INCREMENT PRIMARY KEY,
  pseudo VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
-- 2. Table blagues
CREATE TABLE blagues (
  id INT AUTO_INCREMENT PRIMARY KEY,
  question VARCHAR(500) NOT NULL,
  reponse VARCHAR(500) NOT NULL,
  auteur_id INT,
  likes INT DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (auteur_id) REFERENCES utilisateurs(id) ON DELETE SET NULL
);
-- 3. On remplit
INSERT INTO utilisateurs (pseudo, email) VALUES
('Alice', 'alice@mail.com'),
('Bob', 'bob@mail.com'),
('Toto', 'toto@mail.com');
INSERT INTO blagues (question, reponse, auteur_id) VALUES
('Pourquoi les plongeurs plongent en arrière ?', 'Parce que sinon ils tombent dans le bateau !', 1),
('Pourquoi JavaScript est triste ?', 'Parce qu’il a pas de class !', 2),
('Comment appelle-t-on un boomerang qui ne revient pas ?', 'Un bâton.', 3);
Tu as maintenant une base complète avec 2 tables liées et des données !
9. Les commandes que tu vas taper tous les jours
-- Voir tout
SELECT * FROM blagues;
-- Filtrer
SELECT * FROM blagues WHERE likes > 0;
-- Trier
SELECT * FROM blagues ORDER BY created_at DESC;
-- Ajouter
INSERT INTO blagues (question, reponse, auteur_id) VALUES ('Ta blague ?', 'Ta réponse !', 1);
-- Modifier
UPDATE blagues SET likes = likes + 1 WHERE id = 1;
-- Supprimer
DELETE FROM blagues WHERE id = 5;
-- Joindre deux tables
SELECT b.question, u.pseudo
FROM blagues b
JOIN utilisateurs u ON b.auteur_id = u.id;

Explications:

SELECT * FROM blagues;
Ce que ça fait vraiment :
Tu dis à la base : "Montre-moi TOUT ce qu’il y a dans la table blagues". Toutes les colonnes, toutes les lignes, sans exception.
Équivalent Excel : Tu ouvres le fichier et tu regardes tout le tableau, du haut jusqu’en bas.

C’est la commande la plus simple et la plus utilisée pour vérifier que tout est bien là.
SELECT * FROM blagues WHERE likes > 0;
Ce que ça fait vraiment :
Tu dis : "Montre-moi seulement les blagues qui ont AU MOINS 1 like". Les blagues à 0 like sont cachées.
Équivalent Excel : Tu mets un filtre en haut de la colonne "likes" et tu coches "plus grand que 0".

WHERE = le filtre magique. Tu peux mettre =, >, <, >=, <=, !=, LIKE, IN…
SELECT * FROM blagues ORDER BY created_at DESC;
Ce que ça fait vraiment :
Tu dis : "Montre-moi toutes les blagues, mais trie-les par date de création, de la plus récente à la plus ancienne".
Équivalent Excel : Tu cliques sur la colonne "date" → "Trier du plus récent au plus ancien".

ORDER BY = tri
DESC = du plus grand au plus petit (DESCending)
ASC = du plus petit au plus grand (ASCending → par défaut)
INSERT INTO blagues (question, reponse, auteur_id) VALUES ('Ta blague ?', 'Ta réponse !', 1);
Ce que ça fait vraiment :
Tu ajoutes une NOUVELLE blague dans la table. Tu précises quelles colonnes tu remplis et leurs valeurs.
Équivalent Excel : Tu descends tout en bas du tableau et tu remplis une nouvelle ligne.

Tu peux ajouter plusieurs lignes d’un coup :
VALUES ('blague1', 'réponse1', 1), ('blague2', 'réponse2', 2);
UPDATE blagues SET likes = likes + 1 WHERE id = 1;
Ce que ça fait vraiment :
Tu dis : "Prends la blague dont l’id est 1, et ajoute +1 à son nombre de likes".
Équivalent Excel : Tu trouves la ligne, tu vas dans la cellule "likes", tu ajoutes 1.

ATTENTION : Si tu oublies le WHERE, TOUTES les blagues auront +1 like → catastrophe !
Toujours faire un SELECT … WHERE … avant pour vérifier que tu touches la bonne ligne.
DELETE FROM blagues WHERE id = 5;
Ce que ça fait vraiment :
Tu supprimes complètement la blague dont l’id est 5. Elle disparaît pour toujours (sauf si tu as une sauvegarde).
Équivalent Excel : Tu sélectionnes la ligne entière → Supprimer.

Même règle que UPDATE : si tu oublies le WHERE, tu supprimes TOUTE la table → game over.
SELECT b.question, u.pseudo FROM blagues b JOIN utilisateurs u ON b.auteur_id = u.id;
Ce que ça fait vraiment :
Tu mélanges deux tables pour afficher la question + le pseudo de l’auteur en une seule ligne.
Équivalent Excel : Tu fais une rechercheV sur l’id de l’auteur pour aller chercher son pseudo dans l’autre onglet.

b et u = surnoms (alias) pour écrire plus court.
JOIN … ON = "relie les deux tables là où auteur_id = id".

Tu peux faire :
• INNER JOIN ou JOIN → seulement les blagues qui ont un auteur
• LEFT JOIN → toutes les blagues, même si l’auteur est supprimé (pseudo = NULL)
Tu viens de comprendre les 7 commandes que tu vas utiliser 95 % du temps.

Tout le reste (transactions, index, vues…) c’est du bonus.
Tu es officiellement dangereux en SQL
11. Les relations entre tables (en profondeur)
Une relation, c’est le lien entre deux tables. Par exemple, une blague est liée à son auteur.

Clé primaire → le numéro unique de chaque ligne (id).
Clé étrangère → le numéro de l’autre table (auteur_id = id de l’utilisateur).

Types de relations :
- 1 à 1 : un utilisateur a un profil unique.
- 1 à N : un utilisateur écrit plusieurs blagues.
- N à N : plusieurs utilisateurs likent plusieurs blagues → on crée une table intermédiaire (likes).

Avec ON DELETE SET NULL → si l’utilisateur est supprimé, la blague reste mais auteur_id devient NULL.
Exemple dans notre base :
-- Ajoute une table likes (N à N)
CREATE TABLE likes (
  id INT AUTO_INCREMENT PRIMARY KEY,
  utilisateur_id INT,
  blague_id INT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (utilisateur_id) REFERENCES utilisateurs(id) ON DELETE CASCADE,
  FOREIGN KEY (blague_id) REFERENCES blagues(id) ON DELETE CASCADE
);
Ajoute un like : INSERT INTO likes (utilisateur_id, blague_id) VALUES (2, 1);
Affiche les likes : SELECT u.pseudo, b.question FROM likes l JOIN utilisateurs u ON l.utilisateur_id = u.id JOIN blagues b ON l.blague_id = b.id;

10. 10 exercices pour monter en compétences (clairs et fluides)
Exercice 1 : Créer une base de données
Crée une base appelée "ma_base_test".
Exemple expliqué : CREATE DATABASE ma_base_test; → Ça crée un espace vide pour tes tables. Ensuite USE ma_base_test; pour entrer dedans. Résultat : tu as une base prête.
Exercice 2 : Créer 3 tables avec propriétés
Crée 3 tables : utilisateurs (id INT AUTO_INCREMENT PRIMARY KEY, pseudo VARCHAR(50)), blagues (id INT AUTO_INCREMENT PRIMARY KEY, question TEXT), likes (id INT AUTO_INCREMENT PRIMARY KEY, blague_id INT).
Exemple expliqué : Chaque table a un id unique qui s'incrémente auto, et des propriétés comme VARCHAR pour du texte court, TEXT pour du long texte. Résultat : tu as 3 tables prêtes à recevoir des données.
Exercice 3 : Modifier une table
Ajoute une colonne "email" à la table utilisateurs.
Exemple expliqué : ALTER TABLE utilisateurs ADD COLUMN email VARCHAR(100); → Ça ajoute une nouvelle propriété à la table sans toucher aux données existantes. Résultat : ta table est mise à jour sans perte.
Exercice 4 : Mettre à jour des données
Change le pseudo d'un utilisateur avec id = 1 en "SuperToto".
Exemple expliqué : UPDATE utilisateurs SET pseudo = 'SuperToto' WHERE id = 1; → Ça modifie seulement la ligne avec id=1. Résultat : les données sont mises à jour sans affecter le reste.
Exercice 5 : Ajouter des données
Ajoute un utilisateur "Riri" avec email "riri@mail.com".
Exemple expliqué : INSERT INTO utilisateurs (pseudo, email) VALUES ('Riri', 'riri@mail.com'); → Ça ajoute une nouvelle ligne. Résultat : ta table grandit d’une ligne.
Exercice 6 : Supprimer des données
Supprime l'utilisateur avec id = 2.
Exemple expliqué : DELETE FROM utilisateurs WHERE id = 2; → Ça supprime seulement cette ligne. Résultat : la table est nettoyée sans toucher les autres.
Exercice 7 : Filtrer et trier
Affiche les blagues avec plus de 1 like, triées par date.
Exemple expliqué : SELECT * FROM blagues WHERE likes > 1 ORDER BY created_at DESC; → WHERE filtre, ORDER BY trie. Résultat : tu vois seulement ce que tu veux, rangé.
Exercice 8 : Joindre deux tables
Affiche les blagues avec le pseudo de l’auteur.
Exemple expliqué : SELECT b.question, u.pseudo FROM blagues b JOIN utilisateurs u ON b.auteur_id = u.id; → JOIN relie les tables. Résultat : tu vois les données de 2 tables en 1 requête.
Exercice 9 : Sauvegarder et restaurer
Sauvegarde ta base, supprime-la, et restaure-la.
Exemple expliqué : mysqldump -u root blagues_en_folie > backup.sql (sauvegarde), DROP DATABASE blagues_en_folie (supprime), CREATE DATABASE blagues_en_folie puis mysql -u root blagues_en_folie < backup.sql (restaure). Résultat : ta base est en sécurité.
Exercice 10 : Challenge complet
Crée une table "commentaires" liée à blagues et utilisateurs, ajoute 3 commentaires, et affiche-les avec le pseudo et la question de la blague.
Exemple expliqué : Crée la table avec FOREIGN KEY, INSERT 3 lignes, puis SELECT avec JOIN. Résultat : tu as une base liée sur 3 tables.
